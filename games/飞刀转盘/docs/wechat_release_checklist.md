# 微信小游戏上架前检查清单（飞箭射月球）

## 一、必须补齐（不做会卡上架/影响体验）
- 小程序合法合规信息：隐私政策链接、用户协议、客服联系方式。
- 小游戏图标：主图标（建议 `512x512`）和分享缩略图（建议 `5:4` / `1:1` 各一套）。
- 真正加载流程：独立 Loading 场景，显示真实加载进度（不是仅展示背景图）。
- 音频资源：至少包含点击、发射、命中、失败、通关 5 类音效。
- 异常保护：资源加载失败兜底（背景、按钮图、字体）。

## 二、画质/美术建议（当前版本可继续提升）
- HUD 视觉统一：顶部信息条、按钮、标题建议统一一套主题色（蓝金/月光色系）。
- 转盘（月球）细节：可替换成高分辨率月球贴图 + 轻微法线光感，而不只靠矢量绘制。
- 弩与箭：建议改为贴图资源（现在是矢量图形），边缘会更精致。
- 面板背景：开始/失败/通关三张背景建议风格一致，避免“拼接感”。

## 三、交互和留存（建议尽快加）
- 新手引导：第 1 局给一次“点击发射”的动态提示箭头。
- 失败二次激励：失败面板提供“再来一次”主按钮 + “返回首页”次按钮。
- 节奏强化：关卡过关时加短暂镜头缩放和音效，形成“奖励感”。
- 结算页：展示“本局命中率/用时/最高连击”等轻数据。

## 四、性能与机型适配（微信端重点）
- 低端机验证：重点测 30fps 场景（老安卓）。
- 分辨率策略：`Fit Width/Height` 在竖屏机型实测，避免刘海屏遮挡。
- 内存控制：纹理统一压缩，避免一次加载过多大图。
- DrawCall 目标：战斗中尽量控制在低水平，面板打开时也要平稳。

## 五、微信平台专项（发布前）
- 开放数据域与排行榜（如有社交排名需求）。
- 分享能力：自定义分享标题、分享图、落地文案。
- 订阅消息/激励视频（如有商业化规划）。
- 审核素材：准备至少 3 张实机截图 + 玩法说明文案。

## 六、当前项目已完成项（本轮）
- 关卡流程可跑通：失败重开、通关下一关、全关完成后重开。
- 月球转盘、弩发射点、箭矢视觉已接入。
- 顶部关卡/剩余飞箭信息与面板样式已强化。
- 游戏背景、加载背景已接入资源自动加载。

## 七、建议下一步优先级（按投入产出排序）
1. 增加音效与独立 Loading 场景。
2. 增加主图标与分享图资源。
3. 弩/箭/月球改为高质量贴图并统一风格。
4. 增加新手引导与失败激励文案。
